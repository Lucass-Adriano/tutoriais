Introdução

Você está super atrasado(a) para entregar aquele trabalho e dentro de 15 minutos ele não será mais aceito, você digita freneticamente com a intenção de terminar o mais rápido possível. Então, quando faltam apenas 5 minutos... É ele! O império da tela azul ataca novamente!! Você, como infeliz usuário de windows, nada mais tem a fazer senão chorar ou se lamentar. Quando menos pior, você chegou a salvar parte do trabalho...

Esse é apenas um dos motivos para você querer instalar linux, e nem chega a ser o maior! Dependendo da sua profissão, você talvez já saiba de antemão como funcionam as licenças de software proprietário e de software livre.

Contudo, muita gente não sabe, ainda, quais são as vantagens reais do linux. Muita gente, inclusive, chegou a instalá-lo mas não usou ou não viu motivos para usá-lo e voltou para windows. Mas tem uma coisa que não sai da cabeça... Por que quando encontramos alguns usuário de linux que manjam um pouco mais eles são radicalmente a favor do linux e de software livre? O que faz com que essas pessoas queiram instalar linux em seus computadores e rejeitem completamente o windows & cia de suas vidas? Será que são somente motivos técnicos?? Será que é simplesmente por que linux não trava, tem um gerenciamento e memória melhor (e tem mesmo), é mais seguro (e é muito mais mesmo), etc? Mistério...

Esse artigo tentará desmistificar isso. Ao longo desse texto, eu tentarei, da forma mais clara possível, explicar os motivos pelos quais usar linux é tão necessário e tão vantajoso e tentarei servir de guia para os passos a serem seguidos para a instalação e uso do sistema.

Licença

O primeiro grande motivo para se instalar o GNU Linux é a licença. Vamos começar quebrando alguns mitos:
  1. Você pode fazer programas comerciais de código fechado para linux
  2. Existem programas para linux que não são grátis, assim como no windows
  3. Quando você compra um cd de linux na banca de jornal, no supermercado ou em uma loja de informática, você não está pagando pelo linux, está pagando pelo serviço que alguém está prestando por distribuí-lo, mas você pode obtê-lo de outras formas
  4. O Linux é mantido e pessoas se responsabilizam pelo mesmo, não é apenas um amontoado de código solto pela internet

Nesse ponto, muitos devem estar achando normal o que eu disse acima, os mais acostumados com o free software. Outros devem estar achando estranho. O caso é que o Linux foi criado inicialmente pelo finlandês Linus Torvalds, que vendo que não iria terminar o trabalho sozinho, colocou tudo o que ele tinha feito do linux até então e, para que não fizessem mal uso do que ele tinha colocado lá, usou uma licença para essa distribuição. Essa licença é a conhecida GPL, GNU Public Licence ou Licença Pública GNU. GNU é uma instituição que em seu início esteve filiada ao MIT (Michigan Institute of Technology) e o significado da sigla é recursivo: GNU significa GNU is Not Unix.

A frase apenas demonstra que GNU não é uma marca, um trademark, como os outros sistemas UNIX (AIX da IBM, Solaris da Sun, etc.) costumam ser. Muitas outras curiosidades podem ser encontradas no site da GNU e eu recomendo uma visita lá caso você esteja interessado em saber mais sobre isso, mas eu tentarei manter o tom prático do tutorial.

A licença GPL pode ser encontrada na íntegra no mesmo site. Em linhas gerais, ela diz que um código fonte que é lançado com licença GPL reserva os direitos autorais daquele código fonte para o autor que o escreveu (para os mais leigos, código fonte é o texto que os programadores escrevem para criar seus programas, eles criam o texto e usam um compilador para transformar aquele texto escrito em uma linguagem de programação em um programa), embora o código seja aberto. Diz também que o código (assim como o programa não podem ser vendidos e se alguém fizer uma alteração no código, então o código (ou o software) alterado também não poderá ser vendido, pois parte dele é proibida de ser vendida.

A licença ressalta, porém, que embora o um código ou programa lançado com essa licença deva ser sempre grátis e com o código aberto (ter o código aberto significa que você pode alterar o programa para que ele funcione de acordo com as suas necessidades), nada impede que serviços ou produtos anexados ao programa sejam comercializados.

Vejamos um exemplo. Imaginemos que você baixe o source do kernel (núcleo) do linux em www.kernel.org, abra o código, altere o linux para funcionar diferente e queira vender. Você não vai poder, pois desse código alterado, só a parte que você alterou é sua, não é possível vender a parte dos outros, conforme diz a GPL. Imagine outro caso agora em que você queira pegar o linux da internet, configurá-lo de uma forma que rode em vários tipos diferentes de computadores e queira não vender o que você alterou, pois isso continua sendo de graça, mas vender o serviço de gravar aquilo em cdrom e comercializar em bancas de jornais. Isso é perfeitamente permitido e feito pela maioria das distribuições de linux atuais.

Esse modelo de desenvolvimento de software é muito agradável porque as pessoas que desenvolvem softwares podem continuar recebendo pelo seu trabalho, uma vez que elas cobram por seus serviços, mas o trabalho final fica disponível para todo mundo. Veja a diferença que essa licença causa quando falamos, por exemplo, de um processador de textos. Pegando o caso da Microsoft, o processador de textos que a mesma produz é o famoso "Word". Esquecendo momentaneamente o caso da pirataria, se o seu filho quiser usar esse processador de textos para uso escolar, você terá de pagar para a microsoft para usar isso.

Não é um programa que praticamente toda pessoa que tem computador usa? No caso do linux, se existir um processador de textos open source, isto é, de código aberto, com licença GPL, ninguém precisaria pagar para usar. Mas quem paga os programadores? Muitos trabalham como voluntários. Valorize esse trabalho, eles estão trabalhando para que ninguém mais tenha que pagar por um "word" da vida, visto que a maioria já teve que pagar por um ou por algo semelhante. Mas muitos podem ter sido contratados por uma empresa que tenha pago aos programadores para desenvolverem aqueles software para que ela usasse, mas como não teria sentido deixar o código fechado, muitas vezes a empresa distribui o programa com licença GPL. Além de estar ajudando muita gente e ajudando o desenvolvimento do linux, ela estará fazendo publicidade de seu nome.

Existem muitas outras formas de se ganhar dinheiro com o open source e ao mesmo tempo ajudar a comunidade a crescer. Visite a página http://tuxedo.org/~esr/ um dia, ela contém bastante informações úteis sobre isso.

Agora só falta um mito, o que diz que ninguém se responsabiliza pelo linux. Isso não é verdade, como se pode ver pelo que está escrito acima, todo programa feito com licença GPL preserva os direitos autorais de seus autores, embora o programa seja aberto. Mais do que isso, existe uma grande comunidade linux que participa do desenvolvimento de quase todo programa feito para o sistema, sendo que suporte técnico é uma coisa que não falta. O mesmo não se pode dizer com relação aos sistemas proprietários, ainda mais se levarmos em consideração que usuários que usam software pirata não tem suporte, não é?

Apenas uma última nota para finalizar o tópico sobre licenças. O fato de um programa ter um dono, ou seja, um responsável por ele, não implica que o desenvolvedor do programa tenha alguma responsabilidade com relação a possíveis estragos que um programa venha a causar em sua máquina. Muita gente diz que é por isso que é necessário/vantajoso se usar softwares como windows nesse casos, pois o produtor (no caso a Microsoft) se responsabilizaria pelos danos. Isso é uma mentira sem tamanho! Verifique na ajuda do windows a presença do contrato de usuário que você aceita sempre que instala o windows (mas nunca lê? ;-), nesse contrato diz claramente que a Microsoft também não se responsabiliza por tais danos. Esse foi um dos motivos que fizeram com que organizações governamentais norte-americanas parassem de usar windows para aplicações que envolvessem segurança.
Instalação

Bem, agora você já entende um pouco melhor por que linux é tão melhor que o windows. Não tem a ver somente com o fato de ser um sistema melhor tecnicamente, usando linux, quem ganha somos nós, os usuários, que teremos cada vez mais programas melhores e distribuídos de graça para todos, além de termos nossa segurança e privacidade muito melhor respeitada.

Mas não será só esse o motivo, certamente, que o fará instalar o linux em seu computador. Você irá instalá-lo porque precisa usar o computador para satisfazer as suas necessidades e o Linux irá prover o necessário para que isso possa virar uma realidade. O Linux já já disponibiliza de muitos programas feitos para ele (programas de windows não rodam em linux, não como regra geral) e dependendo de qual for a sua necessidade, ele irá satisfazer com sucesso. Alguns exemplos são: Desenvolvimento de aplicativos, uso do computador em um escritório, uso do computador como um servidor, etc. O linux ainda pode ser usado, por exemplo, não só em computadores mas em sistemas embutidos, como TVs, relógios, calculadoras, etc. Mas vamos pular essa parte.

Ao instalar o linux em seu computador, você poderá fazer isso de várias formas, sendo duas as mais comuns, que são instalar linux e windows no mesmo computador ou instalar exclusivamente linux. Quando for instalar linux e windows, sempre que você iniciar o computador aparecerá um menu, perguntando qual o sistema operacional (linux ou windows) que você desejará utilizar. Quando você instalar exclusivamente linux, normalmente haverá esse menu com a única opção sendo o linux e normalmente ele entrará automaticamente.

Instalar linux junto com windows sem formatar o HD (sem perder todos os dados existentes em seu computador primeiro) é uma pouco difícil, mas não impossível. Basicamente, a dificuldade da instalação depende muito de qual distribuição de linux se está tentando instalar. Existem muitas empresas que criam distribuições (com a licença discutida acima) para fácil instalação, disponibilizam tudo via internet mas criam também cds para serem distribuidos e vendem serviços de suporte técnico, instalação, etc. Esse é o caso da Conectiva e TurboLinux, por exemplo, uma distribuição/empresa brasileira. Também é o caso de algumas distribuições bem conhecidas como RedHat e Mandraque por exemplo.

Um dos endereços onde se pode ter acesso a todo o conteúdo dessas distribuições é o rufus: http://rufus.w3.org/linux/RPM/. Confira no site, é possível pegar tudo sem comprar um cdrom, caso alguém duvide. Normalmente, essas distribuições vem com manuais simples de instalação, que também é muito simples. Não entrarei em detalhes sobre como proceder nos detalhes da instalação, já existe bastante documentação falando sobre isso, vou me concentrar mais em oferecer ao leitor desse texto uma nova visão daquilo que ele(a) sempre deve ter ouvido tanto falar.

Embora grande parte das distribuições de linux sejam feitas por empresas, existe uma distribuição especial que é mantida somente por usuários voluntários, filiados a uma instituição sem fins lucrativos chamada Debian. Essa é a distribuição que eu prefiro e a que eu recomendo, mas muitos usuários acham que sua instalação é muito complicada. Para quem pensa dessa forma, recomendo a compra de um cd da distribuição Mandraque, que é a que tem apresentado melhores resultados. Para os que desejam tentar instalar o Debian, dou meus parabéns e apresento-lhes o seguinte site: http://www.Debian.org/releases/stable/i386/install.pt.html.

Nele você pode encontrar instruções detalhadíssimas, em português, para instalação dessa distribuição de linux nos mais variados tipos de máquinas e das mais diferentes formas (instalação por cdrom, por disquete, via, internet, etc). Lá tem tudo o que você precisa para conseguir instalar o seu linux.

Utilização

Você já foi naquele site ou já comprou o cdrom e já instalou o linux em sua máquina! E agora? Como usar? O que é isso na sua frente? O que você faz com isso? Você pode fazer muitas coisas, basta descobrir isso... Existe muita documentação boa ensinando a usar o linux, qualquer que seja a sua necessidade. O primeiro site que você deveria visitar é o linuxdoc: http://www.linuxdoc.org. Se você tem problemas com inglês, procure no site por "Translated guides", grande parte dos documentos estão traduzidos para várias línguas.

A primeira coisa que você deve saber é que depois de instalado o linux pode vir com login gráfico ou não. Como assim? Se você instalou o linux, deve saber que são criados vários usuários no sistema que podem usar o computador. Por exemplo, se a sua irmão chata não para de ficar mexendo nas suas configurações, crie uma conta pra ela, chamada "chata" (só sugestão). Sempre que ela quiser mexer no computador, ela digita "chata" e em seguida digita a sua senha. Pronto, ela está utilizando e se ela alterar alguma configuração, serão apenas as configurações dela, as suas ficam na sua conta. Esse recurso é chamado de multi-usuário.

Outra coisa importante é que o linux, diferente do windows, pode ser usado em dois modos: modo gráfico e modo texto. O modo texto seria o semelhante a estar no modo DOS do windows, para quem conhece, mas ainda assim é bem diferente, pois mesmo em modo texto é possível rodar vários programas ao mesmo tempo e a partir de várias contas (multi-usuário).

São essas e outras diferenças pequenas que você terá de se acostumar para usar linux. No linux, freqüentemente é necessário, dependendo dos programas que você pretender usar, o modo texto para executar os programas. Também é necessário saber que dispositivos como cdrom, disquete e outros dispositivos removíveis precisam ser montados antes de serem utilizados, diferente do windows em que você só precisa trocar o dispositivo. Todas essas pequenas dicas de utilização podem ser encontradas em manuais no linuxdoc ou em vários outros locais da internet, existem muitos materiais de linux dissertando sobre o assunto.

Nesse ponto, muitos usuários podem achar que linux é mais difícil de ser utilizado que windows. A questão aqui é o motivo pelo qual se compra um computador, se para ser utilizado como máquina de escrever ou se comprado para ser utilizado para tarefas onde realmente se faz necessário o uso de um computador. Muitas distribuições de linux, dentre elas a Conectiva e a Redhat, criam instalações tão simples que o usuário mal precisa saber o que é um computador para instalar linux nele. Uma distribuição, a SUSE, criou um sistema de instalação em que você fica jogando TÉTRIS enquanto a instalação acontece. O mesmo acontece na hora da utilização, normalmente.

Se você está migrando do windows para o linux e era um usuário experiente em windows, o que mais irá atrapalhá-lo serão os vícios que você tinha do sistema operacional antigo. Contudo, se você é um usuário que não sabe mexer muito bem em um computador ainda, nem em linux e nem em windows, então vai se sentir bem mais a vontade com o linux.

O fato é que para ser vantagem utilizar linux hoje, é realmente necessário aprender alguns comandos a mais, sendo que isso pode ser mais fácil ou mais difícil dependendo do tipo de usuário que você seja. Por mais incrível que pareça, usuários menos experientes parecem ter mais facilidade em aprender a usar linux.

Se você tentar instalar e usar linux de fato, talvez perca algum tempo com isso, de fato, mas o que esse artigo tenta passar é que isso vale a pena no final. Quando você puder considerar a opção de usar aplicativos gratuitos ao invés de proprietários, vai entender o que quero dizer. Melhor ainda, quando você se habituar um pouco mais a trocar informações com a comunidade linux, a pedir ajuda, a fazer críticas e perceber que linux não é só um sistema operacional, mas sim uma filosofia e um modo todo novo de se construir, comercializar e distribuir software, nunca mais irá suportar a idéia de colaborar com o aumento do império da tela azul.
